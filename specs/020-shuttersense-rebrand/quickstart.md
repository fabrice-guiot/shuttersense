# Quickstart Guide: ShutterSense.ai Application Rebrand

**Feature**: 020-shuttersense-rebrand
**Date**: 2026-01-17

## Prerequisites

Before starting the rebrand implementation:

1. **Logo Assets Ready**: Ensure you have the following logo files prepared:
   - SVG version of the ShutterSense.ai logo (for sidebar)
   - High-resolution PNG (512x512+) for login page 3D logo
   - Source file that can be exported to multiple favicon sizes
   - Open Graph image (1200x630) for social sharing

2. **Development Environment**: Standard development setup with:
   - Node.js and npm for frontend
   - Python 3.10+ for backend/CLI
   - Git for version control

## Implementation Order

### Step 1: Add Logo Assets (First)

```bash
# Create the public directory if it doesn't exist
mkdir -p frontend/public

# Add your logo files:
# - logo.svg (sidebar, scalable)
# - logo.png (sidebar fallback, 200x50 / 400x100 @2x)
# - logo-login.png (login page, 512x512+)
# - logo-login.webp (login page optimized)
# - favicon.ico (16/32/48 multi-res)
# - favicon-192.png (192x192, Android Chrome)
# - apple-touch-icon.png (180x180, iOS)
# - favicon.svg (scalable, modern browsers)
# - og-image.png (1200x630, social sharing)
```

### Step 2: Update Frontend Core Files

1. **frontend/index.html**:
   - Update `<title>` to "ShutterSense.ai"
   - Add favicon links in `<head>`
   - Add Open Graph meta tags

2. **frontend/package.json**:
   - Update `"name"` to `"shuttersense-frontend"`
   - Update `"description"` to include ShutterSense.ai

3. **frontend/src/components/layout/Sidebar.tsx**:
   - Update logo display to use new logo from public/

4. **Login page component**:
   - Add 3D logo display

### Step 3: Update Backend Configuration

1. **backend/src/main.py**:
   - Update FastAPI app `title` and `description`

2. **Environment variable files**:
   - `backend/.env.example`
   - `backend/src/utils/crypto.py`
   - `backend/src/utils/logging_config.py`
   - `backend/src/utils/security_settings.py`

   Pattern: `PHOTO_ADMIN_*` → `SHUSAI_*`

### Step 4: Update CLI Tools

1. **version.py**:
   - Update `PHOTO_ADMIN_VERSION` → `SHUTTERSENSE_VERSION`

2. **CLI tools** (photo_stats.py, photo_pairing.py, pipeline_validation.py):
   - Update help text and descriptions

3. **templates/base.html.j2**:
   - Update footer to "Generated by ShutterSense.ai"

### Step 5: Update Documentation

1. **README.md**: Complete rewrite with:
   - ShutterSense.ai branding
   - Tagline: "Capture. Process. Analyze."
   - Production URL: https://app.shuttersense.ai

2. **CLAUDE.md**: Search/replace all occurrences

3. **docs/*.md**: Search/replace all occurrences

4. **CHANGELOG.md**: Update header

### Step 6: Verification

```bash
# Run backend tests
cd backend && python -m pytest tests/ -v

# Run frontend tests
cd frontend && npm test

# Verify no old branding remains
grep -r "photo-admin" --include="*.py" --include="*.ts" --include="*.tsx" --include="*.md" --include="*.json" --include="*.html" . | grep -v node_modules | grep -v ".git"
grep -r "PHOTO_ADMIN" --include="*.py" --include="*.ts" --include="*.tsx" . | grep -v node_modules | grep -v ".git"
```

### Step 7: Repository Rename (Manual)

After all changes are merged to main:

1. Go to GitHub repository Settings
2. Navigate to General > Repository name
3. Change "photo-admin" to "shuttersense"
4. Update repository description: "ShutterSense.ai - Capture. Process. Analyze."
5. Upload new social preview image (og-image.png)

## Troubleshooting

### Tests failing after rebrand

1. Check for hardcoded environment variable names in test fixtures
2. Ensure `.env.test` or test configuration uses new variable names
3. Run `grep PHOTO_ADMIN` to find any missed occurrences

### Logo not displaying

1. Verify files are in `frontend/public/` (not `frontend/src/assets/`)
2. Check file names match exactly (case-sensitive)
3. Clear browser cache
4. Verify Vite is serving public directory correctly

### Favicon not updating

1. Clear browser cache (hard refresh: Ctrl+Shift+R / Cmd+Shift+R)
2. Check favicon links in index.html are correct
3. Verify favicon.ico contains all required sizes (16, 32, 48)
