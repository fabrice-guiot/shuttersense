# Research: ShutterSense.ai Application Rebrand

**Feature**: 020-shuttersense-rebrand
**Date**: 2026-01-17

## Overview

This document captures the research findings for the ShutterSense.ai rebrand. Since this is primarily a naming/branding change with no new technical implementations, the research focuses on identifying the complete scope of changes needed.

## Research Tasks

**Decision**: 123 files require updates with ~197 branding reference occurrences

**Findings**:

| Category | File Count | Priority | Notes |

| Backend Config | 8 | P1 | Environment variables in utils/ |

| Documentation | 70+ | P2 | README, CLAUDE.md, docs/ |

| Logo/Assets | 9 (new) | P1 | frontend/public/ |
| Tests & Specs | 30+ | P3 | Internal documentation |

**Rationale**: Comprehensive grep search across all source files identified every instance of branding references.

**Alternatives Considered**: None - full scope analysis is required for complete rebrand.

### 2. Logo Asset Requirements

**Decision**: 9 logo asset files needed in frontend/public/

**Findings**:

| File | Format | Dimensions | Purpose |
|------|--------|------------|---------|
| `logo.svg` | SVG | Scalable | Primary sidebar logo |
| `logo.png` | PNG | 200x50 (400x100 @2x) | Sidebar fallback |
| `logo-login.png` | PNG | 512x512+ | Login page 3D logo |
| `logo-login.webp` | WebP | 512x512+ | Login page (optimized) |
| `favicon.ico` | ICO | 16/32/48 multi-res | Legacy browsers |
| `favicon-192.png` | PNG | 192x192 | Android Chrome |
| `apple-touch-icon.png` | PNG | 180x180 | iOS |
| `favicon.svg` | SVG | Scalable | Modern browsers |
| `og-image.png` | PNG | 1200x630 | Social sharing |

**Rationale**: Industry-standard favicon and logo requirements for cross-browser/platform support.

**Alternatives Considered**:
- WebP-only favicons: Rejected due to Safari compatibility
- Single PNG favicon: Rejected for poor scaling on high-DPI displays

### 3. Environment Variable Migration Strategy

**Decision**: Direct replacement (no backward compatibility layer)

**Environment Variables to Update**:

| Old Name | New Name | Location |
|----------|----------|----------|
| `PHOTO_ADMIN_DB_URL` | `SHUSAI_DB_URL` | backend/.env.example |
| `PHOTO_ADMIN_MASTER_KEY` | `SHUSAI_MASTER_KEY` | crypto.py |
| `PHOTO_ADMIN_LOG_LEVEL` | `SHUSAI_LOG_LEVEL` | logging_config.py |
| `PHOTO_ADMIN_LOG_DIR` | `SHUSAI_LOG_DIR` | logging_config.py |
| `PHOTO_ADMIN_ENV` | `SHUSAI_ENV` | logging_config.py |
| `PHOTO_ADMIN_AUTHORIZED_LOCAL_ROOTS` | `SHUSAI_AUTHORIZED_LOCAL_ROOTS` | security_settings.py |
| `PHOTO_ADMIN_SPA_DIST_PATH` | `SHUSAI_SPA_DIST_PATH` | security_settings.py |
| `PHOTO_ADMIN_VERSION` | `SHUTTERSENSE_VERSION` | version.py |

**Rationale**: Photo Admin was never officially released (pre-release only), so no backward compatibility is needed. Clean break simplifies codebase.

**Alternatives Considered**:
- Dual support with deprecation warnings: Rejected - unnecessary complexity for pre-release software
- Environment variable aliasing: Rejected - adds technical debt

### 4. Frontend Component Updates

**Decision**: Update Sidebar.tsx and LoginPage.tsx for logo display

**Findings**:

Current state:
- `frontend/index.html` - Title is "Photo Admin", no favicon configuration
- `frontend/src/components/layout/Sidebar.tsx` - May have hardcoded app name
- Login page needs 3D logo placement

Required changes:
1. Add favicon links to index.html `<head>`
2. Update `<title>` to "ShutterSense.ai"
3. Add Open Graph meta tags
4. Update sidebar to use `logo.svg` from public/
5. Update login page to display `logo-login.png` (with WebP fallback)

**Rationale**: Standard React/Vite pattern for static assets in public/ directory.

### 6. Documentation Update Strategy

**Decision**: Batch search/replace with manual review for context-sensitive updates

**Files requiring full rewrite**:

**Files requiring search/replace**:
- `CLAUDE.md` - 40+ occurrences
- `CHANGELOG.md` - Header only
- `docs/*.md` - All documentation files

**Search/Replace Patterns**:
```
photo-admin → shuttersense (URLs, packages)
photo_admin → shuttersense (Python)
PhotoAdmin → ShutterSense (classes)
PHOTO_ADMIN → SHUSAI (env vars)
Photo Admin → ShutterSense.ai (display)
```

**Rationale**: Systematic approach ensures completeness while allowing manual review for edge cases.

---

### 7. Repository Rename Process

**Decision**: GitHub rename is a manual owner action after code changes

**Process**:
1. Complete all code/documentation changes
2. Merge to main branch
3. Owner navigates to Settings > General > Repository name
4. Change from "photo-admin" to "shuttersense"
5. GitHub automatically sets up redirect from old URL
6. Update repository description with tagline
7. Upload new social preview image

**Rationale**: GitHub handles redirects automatically. Code changes should be complete before rename to avoid confusion.
